DevRoulette by pavlobaron

DevRoulette is a game for geeks. You log in, tell it what your programming skills are. Then you get a Co-geek, and you two have 20 minutes to solve a task together. If you do, you both earn points. If not - no points - next task. That's it.

  • you tell the platform what you can in terms of programming languages
  • you say "Start"
  • the platform assigns you a random partner (Co)
  • you two get a task from a random pool. You never get a task you have previously solved. Previously unsolved tasks can be reassigned
  • you have 20 minutes to solve this task together by writing code
  • you can communicate through chat or / and code, bu only one person at a time can be the driver
  • you can test the code through running it and checking the result
  • when the time is over, you have no more chance - the code will be finally executed. If the result is correct, you both earn points. If not, you both have failed
  • from here, you can start a new task. You will be assigned to another random task / Co and so on. Or you can log out

Sandboxing

Per game session there will be one sandbox. Since it will be necessary to protect the host from hackers who will try to kill it instead of solving their tasks, a virtualization based approach is needed. When a session starts, a virtual machine is started or assigned from pool. It has no connection to the outer world. After 20 minutes it dies, no matter what, and only runs the final test to check if they have solved the task.
